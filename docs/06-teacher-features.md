# 06 教师端功能（微信小程序）

> **作用**：定义教师端小程序的页面、交互细节与验收点。
> **读者**：教师端 agent（`app/src/pages/*` 负责人）。
> **编辑边界**：可调整组件实现细节；账号绑定流程、借物车交互方式与签名方案为已锁决策。

## 6.1 页面总览

| 页面 | 路径 | 说明 |
|------|------|------|
| 登录 | `/pages/login/index` | 账号密码 + 绑定 openid |
| 首页 | `/pages/index/index` | 看板 / 通知 / 常用功能 |
| 借用主页 | `/pages/borrow/index` | 搜索 + 借物车 + 提交 |
| 借用申请表 | `/pages/borrow/form` | 填写用途、归还日期、签名 |
| 凭证 | `/pages/borrow/voucher` | 借用详情 / 凭证页 |
| 归还 | `/pages/return/index` | 选择借用单归还 |
| 我的 | `/pages/profile/index` | 账号信息、凭证列表、退出登录 |

## 6.2 登录与 openid 绑定（已锁流程）

```
首次：
[用户名] + [密码] → auth.teacherLoginByPassword (传 wx.login 的 code)
   → 后端验证账密 → 返回 token、profile
   → 引导按钮："绑定微信" (button open-type="getPhoneNumber")
   → auth.teacherBindOpenid (phone_code) → openid + phone 写入 ams_teacher

后续：
进入 app → wx.login 拿 code → auth.teacherLoginByOpenid
   → 若已绑定 openid，直接登录返回 token
   → 若未绑定（更换设备）→ 引导走账密流程
```

### 验收点

- 账号密码错误时显示明确提示，不暴露"账号不存在"细节（统一提示）。
- 绑定按钮使用微信 `getPhoneNumber` 组件（小程序合规要求）。
- 同一 openid 不可绑定多个教师账号；同一教师只能绑定一个最新 openid。
- 登录态本地存储（uni.setStorageSync），并设置过期时间。

## 6.3 首页

布局自上而下：

1. **欢迎条**：左侧问候语 + 教师姓名 / 部门，右侧消息小红点。
2. **看板卡片**（教师视角）：
   - 我的在借数量
   - 我的待审批
   - 我的本月借用次数
3. **通知公告**：横向滚动或纵向列表，最多 3 条，点击进详情。
4. **常用功能入口**（九宫格或大图标）：
   - 借用资产
   - 我的借用
   - 归还
   - 凭证
5. **常用资产**（最近借用或推荐）：横向滚动卡片。

**验收点**：

- 下拉刷新所有数据。
- 通知 / 看板加载失败时显示占位。

## 6.4 借用主页（上下分屏 + 点击加入 + 拖拽增强，已锁）

### 6.4.1 布局

```
┌──────────────────────────────────────────┐
│ 搜索栏 [名称/编号 关键词]   [筛选]       │
├──────────────────────────────────────────┤
│ 上半屏：常用 / 搜索结果资产网格          │
│   ┌──────┐ ┌──────┐ ┌──────┐             │
│   │ 图片 │ │ 图片 │ │ 图片 │             │
│   │ 名称 │ │ 名称 │ │ 名称 │             │
│   │ +加入│ │ +加入│ │ +加入│             │
│   └──────┘ └──────┘ └──────┘             │
├──────────────────────────────────────────┤
│ 下半屏：借物车（可折叠 / 拉手把柄）       │
│   - 资产 A   [-] [+] 数量  [×]           │
│   - 资产 B   [-] [+] 数量  [×]           │
│ [清空] [下一步：填写申请表]              │
└──────────────────────────────────────────┘
```

### 6.4.2 主交互

- 搜索：按名称、资产编号、品牌、型号关键词搜索，过滤 `business_status=IDLE`。
- **点击加入**：网格卡片下方"+"按钮直接加入借物车，伴随轻微动画 + 震动反馈。
- **拖拽加强**：长按卡片 ≥300ms 触发拖拽态，使用 `movable-area` / `movable-view` 模拟，松手在下半屏借物车区域时加入。
- 借物车支持调整数量（每个资产可借数量 ≤ `quantity`）。
- 借物车持久化到本地存储，关闭小程序后保留。

### 6.4.3 边界

- 资产不可加入的状态：`LENT` / `PENDING` / `MAINTAIN` / `SCRAPPED` → 卡片置灰并显示原因。
- 借物车为空时禁用"下一步"。

## 6.5 借用申请表

字段（参考 `data/资产借用登记表.md`）：

| 字段 | 类型 | 说明 |
|------|------|------|
| 借用人 | 自动 | 来自当前登录教师 |
| 联系电话 | 自动 | 教师绑定的手机号 |
| 资产明细 | 自动 | 借物车内容 |
| 用途 | 单选 | 教学 / 科研 / 其他（其他需备注） |
| 拟归还日期 | 日期 | 不可早于今天 |
| 备注 | 文本 | 可选 |
| 教师签名 | canvas | 必填 |

**签名组件**：

- 使用 canvas 手写，提供"清空 / 撤销 / 确认"按钮。
- 确认后将签名转 PNG，上传到云存储 `signature/{teacher_id}/{serial_no}.png`。
- 上传成功后调用 `borrow.submit`，传入资产明细 + 签名 fileID。

**验收点**：

- 必填项校验前移（按钮置灰）。
- 网络失败时签名图片本地暂存，避免重画。

## 6.6 凭证页

展示内容：

- 流水号、生成时间。
- 教师姓名、联系电话、用途、拟归还日期。
- 资产明细表（编号、名称、品牌、型号、数量、地点、单价）。
- 教师签名图片。
- 审批管理员、审批时间。
- 二维码（编码 `voucher_qr_payload`，管理员扫码可调用接口校验）。

**操作**：

- 长按截图保存。
- "归还此单" 按钮（仅状态 `APPROVED` 时显示）。

## 6.7 归还

- 列表：当前教师 `status=APPROVED` 的借用单。
- 点击进入归还确认页：列出资产，复核数量，点击"确认归还"。
- 调用 `borrow.return`：状态变 `RETURNED`，资产 `LENT→IDLE`，写日志。

> 一期归还为整单归还；部分归还放二期。

## 6.8 我的

- 教师信息：姓名、部门、手机号、绑定状态。
- 凭证列表：所有借用申请按时间倒序，状态标签可筛选。
- 修改密码、退出登录。

## 6.9 通用 UX

- wot-ui v2 组件库统一风格；主色 `#0096C2`，辅 `#006B8F`。
- 禁用 emoji；使用 wot-ui 内置图标或 iconfont。
- 全局 Loading 与 Toast 走 wot-ui。
- 所有云函数错误统一 Toast 提示 `message`。
