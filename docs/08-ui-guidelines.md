# 08 UI 规范

> **作用**：统一两端的视觉、组件库选型、页面信息密度。
> **读者**：所有前端 agent、设计协作者。
> **编辑边界**：主色 / 辅色 / 无 emoji 为硬约束；其余可调。

## 8.1 主题色

| 用途 | 色值 |
|------|------|
| 主色 | `#0096C2`（BJUT 蓝） |
| 辅助色 | `#006B8F` |
| 文本主色 | `#1F2937`（接近 gray-800） |
| 次要文本 | `#6B7280` |
| 边框 / 分隔 | `#E5E7EB` |
| 背景 | `#F9FAFB` |
| 成功 / 警告 / 错误 | 沿用组件库默认（如 Tailwind `green-600` / `amber-500` / `red-500`） |

> 主色用于按钮、链接、激活态、图表主体；辅助色用于 hover、深色背景、图表第二色。

## 8.2 字体与间距

- 字体：跟随系统默认（管理端 Tailwind `font-sans`；小程序遵循微信默认）。
- 标题层级：H1 24px / H2 20px / H3 16px / 正文 14px / 辅助 12px。
- 行高：正文 1.5；标题 1.3。
- 间距：使用 4 的倍数（4 / 8 / 12 / 16 / 24 / 32）。

## 8.3 组件库

| 端 | 库 | 备注 |
|----|----|------|
| 管理端 | Tailwind CSS + DaisyUI | DaisyUI 主题需扩展自定义色（详见 8.4） |
| 教师端 | wot-ui v2 + UnoCSS | 优先使用 wot-ui 提供的组件 |

### 8.4 管理端主题扩展（DaisyUI 配置示意）

在 `admin/tailwind.config.js` 中扩展自定义主题：

- `primary` = `#0096C2`
- `primary-focus` = `#006B8F`
- 其余使用 DaisyUI 默认。

> 具体扩展实现交给 agent，但变量名固定为 `primary` / `primary-focus`，确保跨模块复用。

### 8.5 教师端主题扩展（wot-ui v2）

- 通过 wot-ui 提供的 CSS 变量覆盖：`--wot-color-primary: #0096C2`。
- 在全局 `uni.scss` 设置统一变量，避免页面级覆盖。

## 8.6 图标

- 管理端：使用 `lucide-vue-next` 或 DaisyUI 内置 SVG；统一线性风格，描边 1.5。
- 教师端：使用 wot-ui 内置图标集；不足时通过 iconfont 引入。
- **禁止 emoji**：任何文本、按钮、提示中不得出现 emoji。

## 8.7 页面信息密度

- 表格：每行 48px，行高足够；密集模式 36px 作为可选。
- 表单：标签在左，输入框靠右；移动端纵向堆叠。
- 卡片：圆角 8px，阴影柔和（如 `box-shadow: 0 1px 2px rgba(0,0,0,.06)`）。
- 列表卡片间间距 12px。

## 8.8 状态标签建议色

| 业务状态 | 标签色 |
|----------|--------|
| `IDLE` | gray |
| `IN_USE` | blue（主色淡） |
| `PENDING` | amber |
| `LENT` | primary（`#0096C2`） |
| `MAINTAIN` | orange |
| `SCRAPPED` | red |

## 8.9 反馈与提示

- 所有写操作完成后 Toast 提示成功 / 失败。
- 不可逆操作（删除 / 报废）必须二次确认弹窗。
- 网络异常统一展示空状态 + 重试按钮。

## 8.10 可访问性

- 颜色对比度满足 WCAG AA（正文与背景 4.5:1）。
- 按钮 / 链接需有清晰的 hover / focus 态。
- 表单输入必须有 label。
