# 01 项目概览

> **作用**：定义 AMS 的目标、角色、范围与技术选型。
> **读者**：所有项目成员，新加入的 agent 第一篇必读。
> **编辑边界**：范围 / 技术栈调整需同步更新，但不展开具体实现细节（细节见 03-04 章）。

## 1.1 项目目标

为学校（以北工大物联网工程中心为典型场景）提供一套覆盖资产生命周期的管理系统：

1. **管理端（Web）**：管理员对学校资产进行增删改查、变动、借出归还、看板分析与报表生成。
2. **教师端（小程序）**：教师通过微信小程序自助借用 / 归还资产，提交申请后由管理员审批。
3. **统一数据源**：使用腾讯云开发（CloudBase）JSON 数据库，前后端共享同一份字段定义。

## 1.2 角色与权限

| 角色 | 客户端 | 主要能力 |
|------|--------|---------|
| 管理员（Admin） | 管理端 Web | 资产 CRUD、入库、变动、借用审批、归还、看板、报表、通知公告、大型资产管理 |
| 教师（Teacher） | 微信小程序 | 浏览资产、提交借用申请（含手写签名）、查看 / 出示凭证、归还资产 |
| 超级管理员（SuperAdmin） | 管理端 Web | 上述全部 + 管理员账号管理 + 字典管理 + `init` 调用 |

> 一期角色只区分管理员与教师；管理员之间权限平等，任一管理员可审批任意申请。

## 1.3 一期范围（MVP）

**必做**：

- 管理端 & 教师端的账号体系（账号密码 + 教师端 openid 绑定免密）
- 资产入库（单条录入，编号自动按现有规则生成）
- 资产查询（列表、筛选、详情、Timeline）
- 资产变动（位置、使用人等，写入 `ams_asset_log`）
- 借用申请 → 单级审批 → 凭证生成（手写签名 + HTML 渲染）
- 归还（教师端发起）
- 管理端 Dashboard（指标卡、4 张图、通知公告、总账信息、出入仓统计）
- 大型资产页（按金额阈值筛选，展示使用率与借出 Timeline）
- 通知公告（管理端发布，教师端首页展示）
- `init` 云函数：超级管理员、字典、示例教师账号初始化

**预留（只占路由、不实现）**：

- AI 语义报表（接入 LLM 生成查询条件并导出 Excel）
- 闲置资产共享 / 显卡算力共享
- 扫码快捷入库 / 借用
- Excel 批量导入

## 1.4 技术栈索引

| 端 | 模板路径 | 核心栈 |
|----|---------|--------|
| 管理端 | `admin/` | Vue 3 + Vite + Vue Router 4 + Tailwind + DaisyUI + CloudBase JS SDK，详见 `admin/README.md` |
| 教师端 | `app/` | unibest 4.4.0 + uniapp + Vue 3 + TS + Vite + UnoCSS + wot-ui v2 + z-paging，详见 `app/README.md` 与 `app/wot-ui/` 内文档 |
| 后端 | `admin/cloudfunctions/` 或独立目录 | 腾讯云 CloudBase（Node 云函数 + JSON 数据库 + 云存储 + 静态托管） |

## 1.5 关键非功能性要求

- **代码简洁、无冗余**：每个模块只做一件事；避免重复实现。
- **模块解耦**：业务模块（asset / borrow / dashboard / ...）目录独立，允许多人并行修改。
- **多 agent 并行**：模块边界即 agent 协作边界，详见 `09-dev-conventions.md`。
- **页面简洁美观**：无 emoji，主色 `#0096C2`，辅 `#006B8F`，遵循各端组件库规范。
- **可追溯**：所有资产变动写入 `ams_asset_log`，借用流程完整记录于 `ams_borrow_request`。
