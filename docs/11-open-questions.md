# 11 开放问题

> **作用**：记录当前未明确、需二期或后续讨论的事项，避免遗忘。
> **读者**：项目负责人 / 二期规划。
> **编辑边界**：问题解决后从本文移除并把结论写入对应章节。

## 11.1 业务规则

- **大型资产价格阈值**：当前默认 5 万元，存于 `ams_dict.config.large_asset_threshold`。是否需要按类别（设备 / 软件 / 家具）分阈值？
- **借用最长周期**：是否限制拟归还日期距今天的最大天数？逾期是否自动告警？
- **超期未还策略**：是否需要超期短信 / 小程序订阅消息通知？是否影响教师后续借用？
- **资产使用率口径**：大型资产页中"使用率"如何计算（已借天数 / 入库天数？近 N 月借出次数？），需与校方对齐。
- **管理员之间是否互通**：一期任一管理员可审批；是否未来需要按部门分配审批权限？

## 11.2 数据与导入

- **首批资产数据来源**：学校现有 Excel 是否提供完整？是否支持一次性批量导入？（一期不在 MVP，但需要落实数据来源）
- **教师账号导入**：教师账号是手动创建还是从教务系统对接？
- **字典初始内容**：部门 / 国标分类的完整清单需校方提供。
- **资产编号兼容**：现有编号有 `YQJJ` / `WXZC` / `TY` 等前缀，是否还有其他类型？映射规则需校对。

## 11.3 流程扩展（二期）

- **部分归还**：一次借用申请的部分资产归还。
- **借用续期**：教师在到期前申请延期。
- **变动审批**：资产位置 / 使用人变动是否需要审批流？
- **二级审批**：高金额资产是否需要双人审批？
- **签名复用**：教师是否可保存一份"留底签名"，简化每次签字？

## 11.4 拓展功能（已规划但未实现）

- **AI 语义报表**：拟接入 LLM（CloudBase AI 或外部 OpenAI 兼容接口），输入自然语言生成 JSON 查询并导出 Excel。
  - 待定：LLM 提供方、报表导出格式、查询白名单字段。
- **闲置资产共享 / 显卡算力共享**：跨部门借用、空闲时段挂出。
  - 待定：共享对象范围、计费方式、信用机制。
- **扫码入库 / 借用**：小程序扫资产二维码。需先确定二维码生成方案（编号 / 短链 / 加密 token）。
- **Excel 批量导入**：管理端上传 Excel 模板批量入库。
- **资产报废与盘点**：一期只做单条报废，是否需要盘点周期？

## 11.5 非功能

- **数据备份**：CloudBase 数据库的备份策略与恢复演练。
- **审计留存周期**：`ams_asset_log` 是否长期保留？需要归档机制？
- **数据导出合规**：导出报表是否需脱敏（手机号、身份证）？

## 11.6 UI / UX

- **黑暗模式**：是否提供？
- **国际化**：是否需要英文版？（一期不需要）
- **无障碍**：是否针对教师残障群体做额外适配？
